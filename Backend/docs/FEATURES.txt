SMARTSHIP LOGISTICS — FEATURES

A) Authentication & identity
- Login via gateway: /gateway/auth/login
- Signup + OTP flow via gateway:
  - /gateway/auth/signup
  - /gateway/auth/resend-otp
  - /gateway/auth/verify-otp
- Password recovery/reset flows:
  - /gateway/auth/forgot-password
  - /gateway/auth/reset-password
  - /gateway/auth/refresh
- Profile/Users/Roles access via restricted identity routes:
  - /gateway/identity/Profile(/{...})
  - /gateway/identity/Users(/{...})
  - /gateway/identity/Roles(/{...})
  - /gateway/identity/admin/{...}

B) Shipment management
- Create shipments, query shipments, update shipment status and related operations.
- Uses EF Core persistence.
- Publishes integration events to RabbitMQ for downstream consumers.

C) Tracking
- Tracking status & history timeline
- Event-driven tracking updates via RabbitMQ consumer
- Supports anonymous tracking retrieval endpoints (implementation in TrackingController)

D) Document management
- Upload document types:
  - General upload
  - Invoice upload
  - Label upload
  - Customs upload
- Fetch documents by document id or by shipment id
- Delivery proof upload (image + signature) for admins
- Files served through /uploads/*

E) Admin
- Dashboard metrics endpoints
- Exceptions management (resolve/delay/return)
- Hubs management (CRUD)
- Locations management (CRUD)
- Reports endpoints (shipment performance, SLA, revenue, hub performance)

F) Platform / cross-cutting
- Central API Gateway with Swagger aggregation
- JWT authentication + role-based authorization
- Serilog logging
- Docker compose infrastructure for SQL Server + RabbitMQ
