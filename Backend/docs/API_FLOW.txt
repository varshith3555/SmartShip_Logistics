SMARTSHIP LOGISTICS — API FLOW (ROUTES + EXAMPLES)

Note: The Angular app calls the gateway paths; the gateway forwards to internal services.

1) Auth flow
- UI -> Gateway: POST /gateway/auth/login
  - Downstream: IdentityService /api/auth/{everything}
- Token refresh:
  - UI -> Gateway: POST /gateway/auth/refresh

2) Identity management surface (restricted)
Only these are intended to be exposed via gateway:
- /gateway/identity/Profile
- /gateway/identity/Profile/{everything}
- /gateway/identity/Users
- /gateway/identity/Users/{everything}
- /gateway/identity/Roles
- /gateway/identity/Roles/{everything}
- /gateway/identity/admin/{everything}

3) Shipment flow
- UI -> Gateway: /gateway/shipments/{everything}
  - Downstream: ShipmentService /api/shipments/{everything}
Typical operations:
- Create shipment (POST)
- Get shipment by id (GET)
- List “my shipments” (GET)
- Admin list shipments (GET)
- Update status (PUT/PATCH depending on controller action)

4) Tracking flow
- UI/public -> Gateway: /gateway/tracking/{everything}
  - Downstream: TrackingService /api/{everything}
Typical operations:
- Get current tracking status
- Get tracking timeline/history
- Admin create/update tracking events (protected)

5) Document flow
- UI -> Gateway: /gateway/documents/{everything}
  - Downstream: DocumentService /api/documents/{everything}
Known endpoints:
- POST /gateway/documents/upload
- POST /gateway/documents/upload-invoice
- POST /gateway/documents/upload-label
- POST /gateway/documents/upload-customs
- GET /gateway/documents/{id}
- GET /gateway/documents/shipment/{shipmentId}
- GET /gateway/documents/customer/{customerId}
- POST /gateway/documents/delivery-proof/{shipmentId} (ADMIN)
- GET  /gateway/documents/delivery-proof/{shipmentId}

6) Static uploads flow
- Browser/UI -> Gateway: GET /uploads/{everything}
  - Downstream: DocumentService /uploads/{everything}

7) Admin flow
- UI -> Gateway: /gateway/admin/{everything}
  - Downstream: AdminService /api/{everything}
Known endpoints:
- GET /gateway/admin/dashboard
- GET /gateway/admin/exceptions
- PUT /gateway/admin/shipments/{id}/resolve
- PUT /gateway/admin/shipments/{id}/delay
- PUT /gateway/admin/shipments/{id}/return
- CRUD:
  - /gateway/admin/hubs
  - /gateway/admin/locations
- Reports:
  - /gateway/admin/reports/shipment-performance
  - /gateway/admin/reports/delivery-sla
  - /gateway/admin/reports/revenue
  - /gateway/admin/reports/hub-performance

8) Swagger aggregation
- Gateway aggregates service swagger documents using SwaggerForOcelot.
- Each route has a SwaggerKey mapped to a service swagger endpoint URL.
