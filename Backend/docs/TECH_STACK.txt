SMARTSHIP LOGISTICS — TECH STACK

A) Frontend
- Framework: Angular 17
- UI library: Angular Material 17 + CDK
- Language: TypeScript (~5.4)
- Reactive: RxJS (~7.8)
- Dev proxy:
  - ng serve uses proxy.conf.json
  - Proxies /gateway and /uploads to the gateway base URL (http://localhost:5000)

B) API Gateway
- Framework: ASP.NET Core
- Routing/Gateway: Ocelot
- Swagger aggregation/UI: MMLib.SwaggerForOcelot
- Target framework: net10.0

C) Microservices
- Framework: ASP.NET Core Web API
- ORM: Entity Framework Core 8
- Database: SQL Server (EF Core SQL Server provider)
- Auth: JWT-based authentication + ASP.NET Core Authorization (roles used)
- Logging: Serilog (Serilog.AspNetCore + Serilog.Sinks.File)
- Target framework: net8.0

D) Messaging / Integration
- Broker: RabbitMQ
- Client: RabbitMQ.Client
- Pattern:
  - Event publishing via SmartShip.Core.Messaging.IEventBus
  - RabbitMQService publishes to direct exchange smartship_events using routingKey = event type name
  - TrackingService runs a BackgroundService consumer and binds tracking_queue to multiple event types

E) Infrastructure / Dev
- docker-compose:
  - SQL Server 2022 container
  - RabbitMQ management container
  - Gateway + microservices containers (root compose)

F) Documentation/Design notes
- Gateway routes follow the convention:
  - /gateway/<service>/* for API calls
  - /uploads/* for document/static files
- Auth routes are exposed only under /gateway/auth/*


Angular 17, TypeScript, ASP.NET Core (.NET 8/10), Ocelot API Gateway, Entity Framework Core, SQL Server, RabbitMQ, Serilog, Swagger (SwaggerForOcelot), Docker
